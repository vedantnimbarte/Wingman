# Supported providers

All non-Anthropic / non-Gemini / non-ChatGPT / non-Cohere entries share the
`OpenAiCompatProvider` adapter
(`crates/wingman-providers/src/openai_compat.rs`). Add a new hosted
OpenAI-shape clone by extending its `Variant` enum and the mapper functions.

## Connecting and listing models

Run `wingman login <provider>` (or `/login` in the TUI) to probe a key, store
it in the OS keyring, and record the default model — credentials persist
synchronously on login. Every adapter implements `list_models()`, which queries
the provider's API for the models available to your key (an OpenAI-compatible
`GET /models` for compat backends, native endpoints for Anthropic and Gemini).
The TUI `/model` picker uses this to show a live catalog scoped to the providers
you're connected to.

## Enterprise providers

- **AWS Bedrock** ships via the OpenAI-compat surface — set
  `AWS_BEARER_TOKEN_BEDROCK` and adjust the region in `base_url`. The SigV4
  streaming path is not implemented.
- **GCP Vertex AI** uses the OpenAPI endpoint with an OAuth2 access token.
  Populate `GOOGLE_VERTEX_TOKEN` with `gcloud auth print-access-token`.
- **IBM watsonx.ai** is a native adapter — provide `WATSONX_API_KEY` +
  `WATSONX_PROJECT_ID` and it exchanges the API key for an IAM token internally.
