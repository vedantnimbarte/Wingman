# Slash commands

## Built-in

| Command                    | Effect                                                                        |
| -------------------------- | ----------------------------------------------------------------------------- |
| `/help` (`/?`)             | Show the command list.                                                        |
| `/clear`                   | Reset the conversation.                                                       |
| `/login` (`/connect`)      | Connect a provider through the guided wizard.                                 |
| `/logout [provider]`       | Remove a stored API key; no argument logs out the active provider.            |
| `/model [<provider>/<id>]` | Swap the active model live; history preserved. Empty argument opens a picker scoped to your connected providers, listing each provider's live models. |
| `/mode [<name>]`           | Switch permission mode (`read-only`, `plan`, `auto-edit`). Empty opens a picker. |
| `/mcp`                     | Manage MCP servers — add, remove, connect, disconnect live.                   |
| `/add <file>`              | Attach a file to the next prompt.                                             |
| `/usage [clear]`           | Token + cost breakdown by provider; `clear` resets the counters.             |
| `/skills [new <name>]`     | Browse and apply skills; `new <name>` scaffolds a skill.                      |
| `/skill <name>`            | Queue a named skill for the next prompt. `/skill stats [name]` shows usage and outcome counts. |
| `/memory`                  | List saved memories. `/memory forget <name>` deletes one.                    |
| `/recall <query>`          | Search across past sessions for prior context.                               |
| `/learn [status\|reset]`   | Self-learning loop dashboard.                                                |
| `/export [format]`         | Export the transcript (defaults to Markdown).                                |
| `/params`                  | Open the sampling-parameter editor for the active model.                     |
| `/resume`                  | Reopen a previous session for this project.                                   |
| `/find <query>`            | In-transcript search; `/findnext` (`/fn`), `/findprev` (`/fp`), `/findclear` (`/fc`). |
| `/quit` (`/q`, `/exit`)    | Exit wingman.                                                                 |

The file-tree sidebar toggles with `Ctrl+B`: `j`/`k` move, `Tab` descends,
`Enter` inserts the path into the composer. Mouse-wheel scrolling is enabled.

## User-defined commands

Place markdown files at `~/.wingman/commands/<name>.md` (global) or
`<project>/.wingman/commands/<name>.md` (project). When you type
`/<name> rest of line` in the TUI, the markdown body is expanded into the prompt
with the literal token `$ARGS` replaced by `rest of line`, and submitted as if
typed directly. Project-local commands shadow globals.

Example `~/.wingman/commands/refactor.md`:

```markdown
Refactor the following Rust code with these constraints:
1. Keep the public API unchanged.
2. Prefer iterators over explicit loops.
3. Run `cargo clippy` mentally and address obvious lints.

$ARGS
```

Then in the TUI: `/refactor crates/foo/src/lib.rs` expands to a complete prompt.
