# CLI reference

```text
wingman [OPTIONS] [COMMAND]
```

Running `wingman` with no subcommand launches the TUI against the resolved
provider and model.

## Top-level flags

| Flag               | Description                                                          |
| ------------------ | -------------------------------------------------------------------- |
| `--mode <MODE>`    | `read-only` \| `auto-edit` \| `yolo`.                                |
| `--model <MODEL>`  | Model id, optionally prefixed: `anthropic/claude-opus-4-7`.          |
| `--print <PROMPT>` | Run a single prompt and exit (non-interactive).                      |
| `--json`           | Emit newline-delimited JSON events instead of text. Use with `--print`. |
| `-v`, `-vv`        | Increase log verbosity.                                              |
| `--quiet`          | Suppress non-error stderr output.                                    |
| `--version`        | Print version and exit.                                              |
| `--help`           | Print help.                                                          |

## Subcommands

| Command                            | Description                                                       |
| ---------------------------------- | ----------------------------------------------------------------- |
| `config init`                      | Write a starter `~/.wingman/config.toml`. `--force` to overwrite. |
| `config show`                      | Print the merged effective configuration. `--json` for JSON.      |
| `config paths`                     | Print the resolved global and project config paths.               |
| `init`                             | Scan the project and write a starter `WINGMAN.md`.                |
| `checkpoint`                       | Snapshot the working tree into a tagged `git stash`.              |
| `undo`                             | Restore the most recent `wingman checkpoint`.                     |
| `cost`                             | Show per-model token usage and estimated USD spend.               |
| `session list`                     | List recent session JSONL files for this project.                 |
| `session fork`                     | Copy an existing session into a new file (`--at N` truncates).    |
| `worktree create <branch>`         | Create a `git worktree` under `.wingman/worktrees/`.              |
| `worktree list`                    | `git worktree list` passthrough.                                  |
| `worktree remove <path>`           | Remove a worktree by path.                                        |
| `memory export <out>`              | Export the global memory dir to a directory or `.json` pack.      |
| `memory import <path>`             | Import a memory pack (`--force` to overwrite).                    |
| `memory diff <a> <b>`              | Show differences between two packs (or live dir vs. pack).        |
| `review <pr#>`                     | Fetch a PR diff via `gh` and run a one-shot review prompt.        |
| `review-multi`                     | Run a review across multiple `provider/model` reviewers in parallel. |
| `diff <file>` / `diff --patch <p>` | Interactive hunk-by-hunk accept/reject reviewer.                  |
| `discover`                         | Probe localhost for Ollama / LM Studio / vLLM and list models.    |
| `schedule [--all]`                 | Run any `[[schedule]]` entries whose cadence is due.              |
| `skill extract`                    | Mine recent sessions for repeated tool-call sequences.            |
| `pilot run "<goal>"`               | Plan and orchestrate a multi-agent run into a PR.                 |
| `pilot status` / `pilot watch`     | Summary / live dashboard for pilot runs.                          |
